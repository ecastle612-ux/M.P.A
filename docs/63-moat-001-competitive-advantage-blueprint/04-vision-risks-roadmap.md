## 11. Long-term vision

### Where M.P.A. should be

| Horizon | Market position | What competitors struggle to replicate |
| --- | --- | --- |
| **Year 1** | Obvious choice for growing PMs (≈100–1,000 units) who hate training tax; Design Partner–proven OS | Felt speed of Today’s Work + guided lifecycle + supervised AI — not a feature PDF |
| **Year 3** | Default ops OS in the mid-market SMB PM segment; known for “Monday runs itself” | Org-scoped AI memory + automation + team habits + comms graph density |
| **Year 5** | Category reference for **property operations intelligence**; selective expansion into deeper accounting / marketplace | Multi-year operational graph + trust brand (“AI that doesn’t go rogue”) + migration reputation |

### Year 1 (focus)

- Ship adoption gates: payments, screening, e-sign, reliable notifications ([API-003](../48-api-003-background-screening/README.md) · [API-004](../50-api-004-electronic-signatures/README.md) · [API-005](../51-api-005-resident-payments-billing/README.md)).  
- Approve + implement DX-003 P0, DX-004 OS-A/B/C/D, IA-001 IA-A/B/C.  
- Migration story that completes without months of dual entry.  
- Prove 5-Minute Rule on common jobs with Design Partners ([RC-001](../52-rc-001-beta-readiness/README.md)).  

**Not Year 1:** MRI-class GL; autonomous AI; native apps as identity (PWA excellence first).

### Year 3 (expand)

- Portfolio Health + owner narrative excellence.  
- Vendor automation maturity (still confirm-heavy).  
- Offline inspections in field reality.  
- Deeper integrations; selective accounting strength.  
- Measurable retention from AI learning + KB.

### Year 5 (lead)

- Category language: “ops OS,” not “PM software.”  
- Optional marketplace / network effects with privacy design.  
- Competitors can copy screens; cannot copy years of org memory + trust.

### What should remain hard to copy

1. Interrupted-user OS culture across every module  
2. Explainable, human-gated AI that clears real queues  
3. Lifecycle grammar without dual paths  
4. Per-org compounding memory  
5. Migration + enrollment trust earned in production

## 12. Risks

| Risk | Severity | Mitigation |
| --- | --- | --- |
| **Parity distraction** | High | Use this blueprint as filter; Layer 0 gates only |
| **AI autonomy race** | High | Hard L4 gates (IA-001); never trade trust for demos |
| **Moat regression** | High | Dual CRUD paths return → DX audits continuous |
| **Over-claim vs RC-001** | High | Partner messaging matches constrained GO |
| **Accounting buyer mismatch** | Medium | Segment honestly; don’t sell to MRI replacements Year 1 |
| **Incumbent copy of Today’s Work widget** | Medium | Depth: Resolve paths + AI + Inspector, not a list widget |
| **Data hostage perception** | Medium | Clean export; win on ops, not lock |
| **Notification spam from AI** | Medium | Caps + digests (IA-001) |
| **Fair housing / screening AI misuse** | High | Explain-only; human decide (API-003) |
| **Performance at scale** | Medium | Portfolio Health queries + Today’s Work SLAs |

### Prioritization filter (use forever)

Ship / deepen a bet only if it strengthens at least one of:

1. Time-to-clear common jobs  
2. Interrupt recovery  
3. Trust (explainability / human gates)  
4. Org memory / retention  
5. Migration or enrollment stickiness  

If it only improves a competitor comparison spreadsheet, **defer**.
