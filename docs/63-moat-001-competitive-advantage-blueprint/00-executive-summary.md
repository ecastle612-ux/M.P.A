## 1. Executive Summary

Property managers do not switch platforms for another module grid. They switch when **daily work gets faster**, **interrupt recovery gets easier**, and **risk shows up before it becomes a call from an owner**.

M.P.A.’s bet is not “more features than AppFolio.” It is to become the **operating system for portfolio work**:

| Switch trigger (“Why leave?”) | Moat (“Why stay?”) |
| --- | --- |
| Training tax and tribal paths | Zero-Friction + guided lifecycle (DX-003 / WF-003) |
| Dashboard theater, not due work | Today’s Work + Context Actions (DX-004) |
| Chatbot AI that doesn’t clear the queue | Explainable AI Operations Manager (IA-001) |
| Module hopping for one job | Workflow-first architecture + Command Center |
| Painful migrations / dual systems | Smart migration + operational continuity |
| Comms sprawl (text/email/portal) | Unified Communication Hub + QR enrollment |

**Hard truth:** Competitors already win on brand, accounting depth, and ecosystem. M.P.A. wins by making interrupted operators **complete common jobs in ≤5 minutes** and by accumulating **org-specific operational memory** (habits, automations, AI feedback, collaboration) that is expensive to recreate elsewhere.

**What this blueprint refuses:** Feature parity as strategy; autonomous AI that burns trust; copying AppFolio’s dense home.
