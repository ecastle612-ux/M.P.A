## Objective

Define **sustainable competitive advantages** that make M.P.A. the obvious switch for property managers — and hard to leave after adoption.

This package does **not** list features for parity. It answers:

1. Why leave AppFolio / Buildium / DoorLoop / Propertyware / Rent Manager / Rentvine / Yardi Breeze / Avail / TurboTenant / MRI for M.P.A.?  
2. What makes leaving M.P.A. costly once daily work lives here?

**No application code.**

## Binding rules (post-Approve)

1. Prefer deepening **workflow + OS + explainable AI** moats over feature-checklist races.  
2. Table-stakes (payments, screening, e-sign) are **adoption gates**, not the moat — ship them without becoming “another AppFolio.”  
3. Never trade human gates on money/legal/leasing for fake AI autonomy.  
4. Material strategy changes restart Design → Document → Approve.
