# BR-002A — “Looks Great” Standard

**Status:** Approved with BR-002 (2026-07-20)  
**Binding**

## Mindset

Stop thinking like an engineer. Think like a Design Director.

Do **not** ask:

- Is the component correct?
- Is the theme correct?

Ask:

> If this were the first screen a property manager ever saw, would I proudly demo it to a paying customer?

If the answer is anything less than an immediate **yes**, the screen **FAILS**.

## PASS rule (all must be true)

- Brand immediately recognizable  
- Text immediately readable  
- Correct contrast  
- Correct visual weight  
- Balanced spacing  
- Premium alignment  
- Looks intentional  
- Comparable to premium SaaS  

Not merely “technically correct.”

## Design review scores

Every branded surface scores:

| Category | Score |
| --- | --- |
| Readability | /10 |
| Balance | /10 |
| Contrast | /10 |
| Premium Feel | /10 |
| First Impression | /10 |

**Any category below 9/10 = FAIL.**

## Loading screen rule

Never use the full logo / embedded-text artwork as a tiny spinner mark.

Elegant patterns:

```
🏠
Loading your workspace...
```

or

```
🏠
M.P.A.
Loading...
```

Typography (if used) is sharp; the mark is a house symbol — not shrunk wordmark art.

## Mobile drawer rule

Stop scaling the official logo image to carry the wordmark.

Use:

```
🏠
M.P.A.
My Property Assistant
```

Typography is always sharper than shrinking artwork.

## Brand recognition priority

> **Brand recognition takes priority over logo fidelity.**

If shrinking the official artwork makes the brand unreadable, the system must switch to a more appropriate presentation rather than faithfully rendering an unreadable image.

## Success criterion (end of branding sprints)

After BR-002 is implemented and human-certified, logo placement and presentation stop being a recurring issue. Future work focuses on AI, workflows, Design Partner feedback, and capabilities — not whether the logo is visible.
