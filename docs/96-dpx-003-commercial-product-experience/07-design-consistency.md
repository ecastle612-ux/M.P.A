# 07 — Design Consistency

**Package:** DPX-003  
**Status:** Draft — awaiting Approve

---

## Goal

One product — not multiple mini-apps.

## Audit matrix (existing Canopy / shell patterns)

| Element | Check |
| --- | --- |
| Spacing | Consistent rhythm; less dead air |
| Typography | Hierarchy matches shell standards |
| Headers | One-glance entity headers |
| Cards | Prefer fewer; interaction containers only when needed |
| Buttons / toolbelts | Primary vs secondary vs More |
| Search M.P.A. | Same drawer behavior everywhere |
| AI launcher | Same presence / z-index rules (AI-001 / SH-002) |
| Tables / forms | Shared density and labels |
| Drawer / floating actions | Never clip or cover primary content |

## Method

Spot-check all allowlisted surfaces after cognitive-load pass; file residual inconsistencies as friction rows (do not invent new design systems).
