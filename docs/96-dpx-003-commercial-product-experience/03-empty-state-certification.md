# 03 — Empty State Certification

**Package:** DPX-003  
**Status:** Draft — awaiting Approve

---

## Hard rules

1. **No empty widgets** without guidance  
2. **No empty date containers**, timeline bubbles, or schedule cards that look “broken”  
3. If there is no data → **meaningful guidance** (what it means + optional next step)  
4. If the widget provides **no value** when empty → **hide it**  
5. Never display empty UI that implies missing product  

## Copy patterns (examples)

| Context | Guidance |
| --- | --- |
| Maintenance today | “No maintenance scheduled today.” |
| Inspections | “No inspections this week.” |
| Lease renewals | “No lease renewals this month.” |
| Announcements | “No unread announcements.” |
| Messages | “No messages yet. Send the first update to keep everyone aligned.” *(existing tone OK)* |
| Timeline | “No activity yet — updates appear as work happens.” |

## Certification checklist

| Check | Pass |
| --- | --- |
| Dashboard widgets empty → copy or hidden | ☐ |
| Property / resident timelines empty → copy or hidden | ☐ |
| Portal schedule / calendar empties → copy or hidden | ☐ |
| Financial / maintenance lists empty → copy + CTA when useful | ☐ |
| No blank bubbles / zero-height shells | ☐ |
