# Role Journey Documentation

**Status:** Draft for approval  
**Purpose:** Emotional journey for each role — shared product language, distinct feelings.

---

## Shared Journey Spine

Every role moves through the same emotional arc when M.P.A. works:

```
Arrive → Orient → Trust → Act → Confirm → Advance → Rest
```

| Stage | User needs |
|-------|------------|
| Arrive | Low friction entry |
| Orient | Instant understanding of “my world” |
| Trust | Belief the data and status are true |
| Act | One clear primary action |
| Confirm | Certainty it worked |
| Advance | Know what happens next |
| Rest | Permission to stop without dread |

Role differences live in **pace, density, and what “rest” means**.

---

## Property Manager — “Command Without Overwhelm”

### Target feelings
Organized · Confident · Ahead · Prepared · Supported  
**Never:** Overwhelmed · Behind · Guessing what needs attention

### Emotional journey (typical day)

| Phase | Experience | Feeling |
|-------|------------|---------|
| Open app | Operations Console shows ordered attention | Ahead |
| Scan 3 seconds | Know P0 count and top item reason | Oriented |
| Work item | Full context; next action obvious | Capable |
| Hand off | Vendor/tenant/owner clearly notified | Supported |
| Midday | Queue shrinks; progress visible | Momentum |
| End of day | Fewer open P0/P1 than morning — or explicit blockers named | Caught up |

### Emotional risks
- Too many equal-priority items → paralysis  
- Charts instead of actions → false busy-ness  
- Missing context → fear of wrong decision  

### Design commitments
- Home always answers: **What deserves my attention right now?**  
- AI assists triage; PM remains decision-maker on high stakes  
- Density without clutter; keyboard speed without coldness  

### Success memory
> “I closed the morning knowing nothing critical was hiding.”

## Tenant — “Home That Explains Itself”

### Target feelings
Welcome · Safe · Respected · Informed · Independent  
**Never:** Ignored · Confused · Dependent on calling the office for basics

### Emotional journey (living cycle)

| Phase | Experience | Feeling |
|-------|------------|---------|
| Invite / first login | Clear address, lease context, welcome | Welcome |
| Ordinary week | Rent status, maintenance status visible without asking | Independent |
| Problem | Request maintenance in plain language; get timeline | Safe |
| Payment | Pay without shame UX; receipt instant | Respected |
| Waiting | Status updates before they chase | Informed |
| Move-out | Checklist and deposit path explained early | Secure |

### Emotional risks
- Corporate jargon → alienation  
- Silence after a request → anger and phone calls  
- Surprise fees → distrust  

### Design commitments
- Proactive answers for FAQs (rent due, request status, docs)  
- Mobile-first, one job per screen  
- Respectful copy even when overdue  

### Success memory
> “I rarely need to call — the app already told me.”

## Property Owner — “Quiet Certainty”

### Target feelings
Confident · Informed · Profitable · Protected  
**Never:** In the dark · Surprised by spend · Anxious about “what’s going on”

### Emotional journey (relationship cycle)

| Phase | Experience | Feeling |
|-------|------------|---------|
| First access | Properties linked; PM identity clear | Trust begins |
| Monthly | Readable narrative + numbers + exceptions | Informed |
| Expense decision | Clear ask, context, approve/deny | In control |
| Issue period | Maintenance/vacancy explained without spin | Protected |
| Strong month | Performance without hype | Confident / profitable |
| Anytime | Open app → answer “What’s happening with my investment?” | Certainty |

### Emotional risks
- Raw PM operational dumps → overwhelm  
- Delayed or vague reports → churn risk  
- Hidden fees in jargon → suspicion  

### Design commitments
- Executive clarity over operational density  
- Exceptions called out honestly  
- Approvals feel consequential but simple  

### Success memory
> “I always know how my property is doing — without chasing my PM.”

---

## Vendor — “Earn With Dignity”

### Target feelings
Valued · Efficient · Trusted · Rewarded  
**Never:** Ghosted · Buried in paperwork · Unsure about pay

### Emotional journey (job cycle)

| Phase | Experience | Feeling |
|-------|------------|---------|
| Invite / opportunity | Clear job, location, expectations | Respected |
| Accept | Fast accept path | Efficient |
| Perform | Status updates simple from the field | Competent |
| Complete | Evidence upload straightforward | Done |
| Invoice / pay | Payment status visible | Rewarded / trusted |
| Reputation | Fair ratings tied to real jobs | Valued |

### Emotional risks
- Slow PM response after bid → resentment  
- Compliance friction without guidance → drop-off  
- Payment opacity → marketplace exit  

### Design commitments
- Jobs easy to accept, complete, and get paid  
- Blockers (insurance, docs) explained with fix path  
- Field-first: large actions, minimal typing  

### Success memory
> “I did the work and got paid without chasing anyone.”

## Cross-Role Emotional Contracts

| When… | The other party should feel… |
|-------|------------------------------|
| PM assigns vendor | Vendor: trusted with clear scope |
| Tenant submits request | PM: clear intake; Tenant: heard |
| Owner asks a question | PM: context attached; Owner: taken seriously |
| Payment fails | Tenant: guided; PM: alerted without drama |

The platform mediates dignity between sides.

## Journey KPIs (Qualitative)

| Role | Leading experiential signal |
|------|----------------------------|
| PM | “I know what to do next” within 3 seconds of open |
| Tenant | Support tickets for “status questions” decline |
| Owner | “What’s going on?” calls decline |
| Vendor | Time from complete → payment clarity shortens |
