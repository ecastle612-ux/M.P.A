# Micro Interaction Philosophy

**Status:** Draft for approval  
**Purpose:** Every key interaction has an intentional emotional outcome. This is not animation spec — it is experience intent. Motion/visual execution follows **06 Canopy** after approval.

---

## Rule

> Design the feeling first. Then choose the lightest interaction that produces it.

If a micro-interaction does not change confidence, clarity, or momentum, remove it.

---

## Interaction Catalog

### Searching

| Aspect | Definition |
|--------|------------|
| **Intent** | Find something or start an action without hunting |
| **Feeling** | Powerful, fast, in control |
| **Emotional outcome** | “I can reach anything from here.” |
| **Must** | Instant feedback; grouped results; empty query shows useful actions |
| **Must not** | Dump unrelated database rows; feel like raw SQL |

**PM default:** Command palette as the power path.  
**Tenant/Owner:** Simpler search scoped to their world.

---

### Opening a work order

| Aspect | Definition |
|--------|------------|
| **Intent** | Understand and act without assembling context |
| **Feeling** | Prepared — like a briefing landed with the ticket |
| **Emotional outcome** | “I have everything I need to decide.” |
| **Must** | Property, people, urgency reason, history, next action visible |
| **Must not** | Force tab-hopping to learn who lives there |

---

### Paying rent

| Aspect | Definition |
|--------|------------|
| **Intent** | Settle obligation with dignity |
| **Feeling** | Safe, respectful, finished |
| **Emotional outcome** | “I’m square — I have proof.” |
| **Must** | Amount, period, method, confirmation/receipt |
| **Must not** | Shame copy when late; surprise fees at the last step |

Late state: firm and clear, never humiliating.

---

### Signing a lease

| Aspect | Definition |
|--------|------------|
| **Intent** | Commit with understanding |
| **Feeling** | Serious, clear, protected |
| **Emotional outcome** | “I know what I signed and what happens next.” |
| **Must** | Readable document, progress through signatures, post-sign move-in path |
| **Must not** | Rush past review; ambiguous “you’re done” without next steps |

---

### Uploading a document

| Aspect | Definition |
|--------|------------|
| **Intent** | Store proof in the right place |
| **Feeling** | Competent, secure |
| **Emotional outcome** | “It’s filed where it belongs.” |
| **Must** | Show destination entity; progress; success with filename; virus/size errors that teach |
| **Must not** | Silent failure; unclear which property/lease it attached to |

---

### Receiving a notification

| Aspect | Definition |
|--------|------------|
| **Intent** | Learn something that changes a decision or status |
| **Feeling** | Informed, not attacked |
| **Emotional outcome** | “I know why this matters and what to do.” |
| **Must** | Plain title, reason, deep link to entity, priority honesty |
| **Must not** | Badge inflation; vague “You have updates”; night spam for non-critical |

---

### Completing maintenance

| Aspect | Definition |
|--------|------------|
| **Intent** | Close the loop for tenant, PM, vendor, owner visibility |
| **Feeling** | Resolution and relief |
| **Emotional outcome** | “This is done — and everyone who needs to know can see it.” |
| **Must** | Evidence path; status cascade; tenant-readable completion note |
| **Must not** | Leave tenant in “open” limbo; hide invoice from PM workflow |

---

### Finding a vendor

| Aspect | Definition |
|--------|------------|
| **Intent** | Choose capable help with confidence |
| **Feeling** | Supported, decisive |
| **Emotional outcome** | “I picked well — or I know why I’m waiting on bids.” |
| **Must** | Rank with reasons (trade, compliance, performance); clear invite/assign |
| **Must not** | Opaque lists; recommend non-compliant vendors without warning |

## Additional High-Frequency Micros

| Interaction | Feeling target | Emotional outcome |
|-------------|----------------|-------------------|
| Assigning a vendor | Decisive handoff | “Owned by someone capable.” |
| Approving an expense (owner) | Control without friction | “I decided with context.” |
| Sending a message on an entity | Contextual continuity | “This conversation belongs to the job.” |
| Dismissing an AI suggestion | Autonomy | “I’m still in charge.” |
| Clearing last queue item (PM) | Relief | “I can breathe.” |
| Failed payment retry | Guided recovery | “I can fix this.” |
| Invite accept (any role) | Belonging / clarity | “I know what I was invited to.” |

---

## Emotional Outcome Checklist

Before approving any micro-interaction design:

1. What does the user feel *before*?  
2. What should they feel *after*?  
3. What uncertainty was removed?  
4. What is the next step now visible?  
5. Did we add motion/UI without emotional purpose? → remove  

---

## Zero-Learning Micro Rules

| Rule | Application |
|------|-------------|
| Label the verb | Buttons say the outcome (“Assign vendor”, not “Submit”) |
| Show consequences | “Tenant will be notified” before send |
| Keep affordances obvious | Primary action visually singular |
| Prefer recognition | Lists and chips over recall |
| Same pattern everywhere | Opening any entity feels familiar |
