# First Five Minutes Experience

**Status:** Draft for approval  
**Purpose:** Define what each role sees, understands, is surprised by, and delighted by in the first five minutes — the trust window.

---

## Universal First-Five Rules

1. **Value before configuration** — show their world, then ask for setup  
2. **One primary path** — do not present a feature zoo  
3. **Language of their job** — no internal engineering terms  
4. **A win in five minutes** — complete one meaningful action or clearly see next action  
5. **No hostage tours** — optional tips; never block the canvas  

---

## Property Manager — First Five Minutes

### Minute 0–1 — Arrive
**See:** Sign-in → organization context (or create org) → Operations Console shell.  
**Understand:** “This is my command center.”  
**Feel:** Calm density, not empty marketing.

### Minute 1–3 — Orient
**See:** Attention queue (even if empty) with explanation of what will appear; portfolio filter; clear nav groups.  
**Understand:** Home answers what deserves attention.  
**If empty org:** Guided first win — “Add your first property” as the single primary CTA (not 12 setup checkboxes at once).

### Minute 3–5 — First win
**Do:** Create property **or** invite teammate **or** connect a sample/demo attention item (if seeded).  
**Surprise:** Context stays attached; they’re not dumped into a blank CRUD form maze.  
**Delight:** “You’re set up to run operations — here’s what happens next.”

| Should see | Should not see |
|------------|----------------|
| Queue metaphor | Vanity dashboard widgets |
| Next step CTA | Forced 10-step wizard |
| Org name / role clarity | Vendor marketplace admin dump |

## Tenant — First Five Minutes

### Minute 0–1 — Arrive
**See:** Welcome with property address and PM company name; mobile-friendly home.  
**Understand:** “This is my home portal for this lease.”  
**Feel:** Welcome, not corporate onboarding.

### Minute 1–3 — Orient
**See:** Rent status (due/paid), open maintenance (if any), documents entry, messages.  
**Understand:** The three things they care about daily.  
**Proactive answer:** Due date and how to pay visible without hunting.

### Minute 3–5 — First win
**Do:** View lease doc **or** save payment method **or** submit a test-level “how do I…” knowledge answer.  
**Surprise:** Questions answered in-app before they message.  
**Delight:** “You’re all set — we’ll remind you before rent is due.”

| Should see | Should not see |
|------------|----------------|
| Address + rent truth | PM internal queues |
| Big Pay / Request actions | Dense tables |
| Respectful overdue state (if any) | Shame UX |

---

## Property Owner — First Five Minutes

### Minute 0–1 — Arrive
**See:** Portfolio list of *their* properties; PM company branding subtle; last report or “Report in progress.”  
**Understand:** “These are my investments in one place.”  
**Feel:** Executive calm.

### Minute 1–3 — Orient
**See:** Simple nav — Properties, Reports, Approvals, Messages.  
**Understand:** Where performance and decisions live.  
**If first month:** Honest empty — “Your first report publishes on…” with what it will include.

### Minute 3–5 — First win
**Do:** Open a property summary **or** read a sample/first report section **or** set notification preferences.  
**Surprise:** Plain-language performance, not a PM ticket dump.  
**Delight:** “You’ll never have to wonder what’s happening — reports and approvals live here.”

| Should see | Should not see |
|------------|----------------|
| Property performance snapshot | Maintenance triage console |
| Clear approvals inbox | Vendor bid tools |
| Honest gaps | Fake charts with no data |

---

## Vendor — First Five Minutes

### Minute 0–1 — Arrive
**See:** Job inbox (or compliance setup if blocked from jobs).  
**Understand:** “This is how I get work and get paid.”  
**Feel:** Direct, field-ready.

### Minute 1–3 — Orient
**See:** Jobs / Bids / Active / Payments / Profile. Compliance status if incomplete — with exact missing items.  
**Understand:** What blocks earning vs what’s optional.

### Minute 3–5 — First win
**Do:** Complete a compliance item **or** view a job detail and accept/decline path **or** set trade/service area.  
**Surprise:** Payment path explained early (“After completion → invoice → payout”).  
**Delight:** “Accepting and finishing jobs looks fast.”

| Should see | Should not see |
|------------|----------------|
| Clear job cards | Owner financial reports |
| Compliance fix path | PM org settings |
| Pay status concepts | Deep analytics |

## First-Five Delight (All Roles)

Delight is **competence revealed quickly**, not animation tricks:

- The product anticipated their question  
- The next step was obvious  
- Nothing scary happened  
- They feel smarter after five minutes, not dumber  

---

## First-Five Anti-Patterns

| Anti-pattern | Emotional damage |
|--------------|------------------|
| Empty states with no CTA | Abandonment |
| Setup walls before any value | Resentment |
| Same generic dashboard for all roles | Confusion + distrust |
| Interstitial feature marketing | “This is not for me” |
| Asking for 20 profile fields first | Fatigue |

---

## Exit Criteria (First Five)

A role’s first-five design is approved only if:

1. User can state the product’s purpose for them in one sentence  
2. User can identify the next action without help  
3. User completes or clearly schedules one meaningful action  
4. Stress is lower or equal — never higher — than before login
