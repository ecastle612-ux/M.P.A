# 07 — Phasing

**Package:** ADMIN-003  
**Status:** Approved · Slice A unlocked  
**Amendment:** [10-amendment-mission-control.md](./10-amendment-mission-control.md) (binding slice order)

Approve accepts the **full Operations Center vision**. Implementation unlocks **one slice at a time**. Material expansion of a slice restarts Design → Document → Approve for that delta.

---

## Slice A — Headquarters shell (**Unlocked**)

**Goal:** Mission Control home — four information levels — using existing tools. Not a card dashboard.

| Deliverable | Notes |
| --- | --- |
| 1. Immediate Attention | Critical actionable items always visible |
| 2. Business Snapshot | Live KPIs (honest unavailable labels) |
| 3. Operational Workspaces | Grouped deep-links by responsibility (no random cards) |
| 4. Quick Actions | One-click constant jobs |
| Global Search scaffold | Orgs + people roles + properties |
| Rehome existing console | Health, Providers, Flags, Testing, Impersonation under workspaces |
| Product deep-links | Setup, Migration, Ops surfaces |

**Deferred within A:** Executive AI Brief, full workspace density panels, advanced analytics/automation, CRM, Deployments console, inventing deployment status without a signal.

## Slice unlock rule

```
Approve ADMIN-003 (vision + amendments)

    → Explicit unlock required for B / C / D
```

PRs must cite `ADMIN-003 Slice A` (or later slice id when unlocked).
