# 01 — Vision

## Mission

M.P.A. (My Property Assistant) exists to eliminate the operational headaches of property management. We are building the **AI Property Operations Platform** — not another listing tool, not another accounting spreadsheet with a calendar attached, and not a generic dashboard that property managers must bend their business around.

M.P.A. will become the **operating system for property managers**: one connected system where properties, people, money, maintenance, vendors, owners, and communication flow through shared workflows powered by embedded intelligence.

## What M.P.A. Is

| M.P.A. Is | M.P.A. Is Not |
|-----------|---------------|
| An AI operations platform | A chatbot with property screens |
| Workflow-connected end-to-end | A collection of disconnected modules |
| Built for professional PMs at scale | A weekend landlord app |
| Desktop-first, enterprise-grade UX | A generic template SaaS clone |
| A platform vendors and owners connect into | A closed internal tool |

## Long-Term Ambition

M.P.A. will serve **thousands of property management companies** managing portfolios from a handful of doors to thousands. The platform must:

1. **Compress time** — automate repetitive work so PMs focus on decisions and relationships.
2. **Protect revenue** — reduce vacancy, accelerate collections, prevent costly mistakes.
3. **Reduce risk** — compliance, documentation, screening, and audit trails by default.
4. **Connect the ecosystem** — property managers, owners, tenants, and vendors on one operational graph.

## Strategic Pillars

### 1. Workflow Unity
Every screen exists in service of a business workflow. Users never wonder which "module" they are in — they are always moving work forward.

### 2. Vendor Marketplace as Core Infrastructure
Vendors are not an add-on. Work orders, invoices, payments, ratings, and communication with vendors are first-class platform capabilities.

### 3. AI Everywhere
AI is embedded in decisions, drafts, prioritization, matching, and summaries — invisible when it works, explainable when it matters.

### 4. Premium Product Experience
M.P.A. must feel like software a professional pays serious money for. Distinct visual identity. Desktop power. Mobile readiness.

### 5. API-First Durability
The same backend powers web today and native mobile tomorrow. No throwaway frontend architecture.

## Success Metrics (North Star)

| Metric | Why it matters |
|--------|----------------|
| Time to complete core workflows | Validates "save time" promise |
| Occupancy rate improvement | Validates revenue impact |
| Maintenance resolution time | Validates operational efficiency |
| Rent collection cycle time | Validates financial operations |
| Vendor match acceptance rate | Validates marketplace value |
| Owner report engagement | Validates communication quality |
| Platform NPS by persona | Validates holistic experience |

## Non-Goals (v1)

- Replacing full general ledger accounting systems (integrate instead)
- Building in-house payment processing (Stripe Connect)
- Shipping native mobile apps before web PWA is production-grade
- Feature parity with legacy PM software (we win on workflow + AI, not checkbox count)
