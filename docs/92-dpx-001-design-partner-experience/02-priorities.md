# 02 — Priorities (1–9)

**Package:** DPX-001  
**Status:** Draft

Each priority maps to UX-009 amendments and/or DPX-owned artifacts.

---

## Priority 1 — Workflow before features

Every page supports the natural PM workflow.

Per page, document:

| Question | Capture |
| --- | --- |
| Primary goal? | One sentence |
| Top 3–5 actions? | Ordered list |
| Information required immediately? | Above-fold |
| What can be progressive disclosure? | Below / More / Accordion |

**Maps to:** UX-009 Amendments A–B, F · [02-workflow-audit](../88-ux-009-cognitive-load-workflow-optimization/02-workflow-audit.md) · [06-surface-map](../88-ux-009-cognitive-load-workflow-optimization/06-surface-map.md)

---

## Priority 2 — Cognitive load reduction

Pages feel calm. Remove visual competition. 80/20 visibility. Advanced tools never compete with primary actions.

**Maps to:** UX-009 A, F, I · Amendments A–J

---

## Priority 3 — Mobile excellence

Assume the user is at a property. One-hand use. Primary actions visible. Thumb reach, readability, workflow continuity.

**Maps to:** UX-009 G · [07-mobile-standard](../88-ux-009-cognitive-load-workflow-optimization/07-mobile-standard.md)

---

## Priority 4 — Operational AI

Floating AI Copilot is the operational assistant with automatic page context:

| Context | Example |
| --- | --- |
| Resident | Ask about this resident |
| Property | Ask about this property |
| Maintenance | Summarize work order / next step |
| Dashboard | Explain priorities |

AI reduces navigation — it is not another feature to learn.

**Maps to:** UX-009 D · [04-floating-ai-assistant](../88-ux-009-cognitive-load-workflow-optimization/04-floating-ai-assistant.md) · SH-002 store isolation

---

## Priority 5 — Design language

One visual language: one-glance headers, sticky contextual actions, progressive disclosure, consistent spacing/typography/card hierarchy/empty states.

No page should feel like a different application.

**Maps to:** Canopy · UX-009 pattern system · Component standards (12)

---

## Priority 6 — Workflow timing

Measure common tasks (current → target → improvements). Template: [04-workflow-timing.md](./04-workflow-timing.md).

Seed workflows: Create Property · Add Resident · Assign Vendor · Generate Report · Send Announcement · Complete Inspection · Open Lease · Search Resident.

---

## Priority 7 — Eliminate friction

Track unnecessary taps, repeated navigation, confirmations, duplicate info, excessive scrolling, confusing terminology. Registry: [05-friction-registry.md](./05-friction-registry.md).

---

## Priority 8 — First-time experience

A first-time PM is productive without training. Audit empty states, setup, dashboard, search, AI, navigation, help text. Checklist: [06-first-time-experience.md](./06-first-time-experience.md).

---

## Priority 9 — Design Partner certification

Review as a first-time PM. For every workflow: next step obvious? Completable confidently? Faster than competitors? Document hesitation points. Protocol: [07-certification-protocol.md](./07-certification-protocol.md).
