# UX-006 — Mobile Workflow Optimization

**Initiative ID:** UX-006  
**Status:** Design ✔ · Document ✔ · **Awaiting Approve** · Implement locked  
**Constraint:** Presentation / navigation / interaction patterns only — no workflow redesign, no new product features, no business-logic changes  
**Aligns with:** Canopy (06), Experience Architecture (21), UX Principles (07)

---

## Problem

On phone-width viewports, several core surfaces require multi-screen vertical scrolling before operators reach secondary (but frequent) work. Primary actions are sometimes above the fold while related work (timeline, documents, repairs) sits far below. Busy PMs walking properties lose time to scroll hunting.

## Goal

Eliminate unnecessary scrolling and taps while preserving architecture and business workflows. Reorganize information density; keep actions reachable; make AI Ops conversation-first on mobile.

## Non-goals

- New product features or modules  
- Workflow / permission / schema redesign  
- Desktop-first visual redesign  
- Removing information (reorganize / disclose only)
