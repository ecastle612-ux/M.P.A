# Design Partner Guide

**RC-001 · Onboarding & engagement**

## Fit criteria

| Must | Nice |
|------|------|
| 1–3 interested PM orgs | Willing to meet weekly for 30 minutes |
| &lt; 50 units initially | Can supply anonymized sample data |
| Accepts known limitations in writing | Has sandbox willingness for payments/screening |

## Onboarding sequence

1. Kickoff — walk Known Limitations; get written acknowledgment  
2. Admin provisions org + roles ([Administrator Guide](./administrator-guide.md))  
3. Scenario 1 dry-run (new company setup) with partner PM  
4. Scenario 2 dry-run (applicant → pay) in sandbox providers  
5. Optional: Migration Guide dry-run on sample CSV  
6. Weekly feedback for 4–6 weeks  
7. RC exit interview → backlog prioritization  

## Engagement cadence

| Cadence | Activity |
|---------|----------|
| Week 0 | Kickoff + limitations sign-off |
| Weekly | 30-min workflow review |
| Biweekly | Defect triage (P0/P1 only in beta window) |
| Exit | Written GO/NO-GO for wider beta |

## Out of engagement scope

- Building Owner/Vendor portals during beta unless P0 blocker  
- Full GL / QuickBooks  
- Custom one-off integrations  

## Success metrics

- Time-to-first-lease-signed (sandbox)  
- Time-to-first-test-payment  
- # of P0 defects discovered (target: 0 open)  
- Partner NPS / qualitative confidence
