# Beta Guide

**RC-001 · Design Partner Beta**

Welcome to the M.P.A. Design Partner beta. You are helping validate real property-management workflows before open release.

## What you get

- Full PM web operations for small portfolios  
- Applicant screening + lease e-sign rails (sandbox)  
- Resident payments (test/sandbox)  
- Migration import tools  
- Direct feedback channel to product

## What you do not get (yet)

See [Known Limitations](../08-known-limitations.md) — especially Owner/Vendor portals, full accounting, offline sync, and commercial SMS/email.

## How to work with us

1. Use a **sandbox / non-production** Supabase project when possible.  
2. Keep portfolios small (&lt; 50 units).  
3. Report defects with: org name, URL, steps, expected vs actual, screenshot.  
4. Prefer workflow feedback (“I couldn’t finish move-in”) over cosmetic notes first.  
5. Do not put production card data or live bank credentials into beta.

## Success criteria for beta

- You can onboard properties → units → residents  
- You can run maintenance and see Ops Center truth  
- You can collect a test rent payment  
- You can complete at least one applicant → signed lease path  

Thank you for partnering with M.P.A.
